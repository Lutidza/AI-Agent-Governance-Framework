# Регламент проекта документации для ИИ

Этот репозиторий является рабочим проектом по разработке универсальной документации для ИИ-агентов.
Цель проекта: создать единую систему правил, инструкций, карт и workflow для сопровождения и разработки программных проектов.

## Назначение проекта

- Формировать машино-ориентированную документацию для одиночных и мультиагентных сценариев.
- Разделять универсальные правила системы и стек-специфичные блоки.
- Обеспечивать переносимость: в целевом проекте заменяется стековый блок, остальная система остается неизменной.

## Контуры репозитория

- `README.md` (корень): мета-регламент проекта документации, назначения папок и общих правил ведения.
- `AGENTS.md` (корень): операционный контракт для всех подключаемых агентов, работающих в этом репозитории.
- `docs/`: основной продукт проекта.
  - Здесь разрабатывается универсальная система инструкций, правил, карт, примеров и workflow для ИИ в других проектах.
  - Главный индекс внутри контура: [docs/README.md](G:\PacificNET\AI-Workspace\docs\README.md).
- `.aiassistant/`: контур интеграции с продуктами JetBrains.
  - Содержит файлы тонкой настройки среды совместной разработки с ИИ и ИИ-агентами.
  - Этот контур не заменяет универсальные правила из `docs/`, а дополняет их на уровне IDE.

## Правила языка и именования

- Имена файлов и папок: английский язык.
- Содержимое документации: русский язык, если пользователь не указал иное.
- Формулировки: машинные, точные, проверяемые, без двусмысленности.
- Нормативные маркеры: `ОБЯЗАН`, `ЗАПРЕЩЕНО`, `ДОПУСКАЕТСЯ`.
- Условные ветки: формат `IF -> THEN`.

## Разделение областей действия

- [AGENTS.md](G:\PacificNET\AI-Workspace\AGENTS.md) и текущий `README.md` задают правила для агента, который разрабатывает документацию в этом репозитории.
- [docs/README.md](G:\PacificNET\AI-Workspace\docs\README.md) и связанные документы в `docs/` задают правила для применения в целевых программных проектах.
- Эти два контура не конкурируют: они относятся к разным уровням системы.
- Файлы в `.aiassistant/` дополняют процесс на уровне JetBrains-среды и не подменяют правила из `docs/`.

## Базовый процесс изменения документации

1. Зафиксировать, какой контур изменяется: `docs/` или `.aiassistant/`.
2. Подтвердить границы изменения и целевые файлы.
3. Изменить только согласованные файлы.
4. Проверить согласованность терминов, правил и перекрестных ссылок.
5. Кратко зафиксировать результат и назначение изменений.
