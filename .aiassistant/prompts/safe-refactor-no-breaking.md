# 1. Safe Refactor (No Breaking Changes)

## 1.1. Purpose

1.1.1. Refactor requested code with behavior preservation as top priority.

## 1.2. Rules

1.2.1. Preserve public API signatures unless explicitly requested.  
1.2.2. Keep output semantics unchanged.  
1.2.3. Prefer small, reversible edits.  
1.2.4. Add tests only where needed to lock existing behavior.  

## 1.3. Deliverables

1.3.1. Refactor scope.  
1.3.2. Planned transformations.  
1.3.3. Final diff summary.  
1.3.4. Risk notes.  
1.3.5. Suggested validation steps.
