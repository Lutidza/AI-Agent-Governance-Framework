# 1. Generate Commit Message (Standard)

## 1.1. Purpose

1.1.1. Generate a commit message from staged changes.

## 1.2. Requirements

1.2.1. Use one-line subject in imperative mood.  
1.2.2. Keep subject length <= 72 characters.  
1.2.3. Add body with:
1.2.3.1. What changed.  
1.2.3.2. Why it changed.  
1.2.3.3. Risk/compatibility notes.  
1.2.4. Avoid vague wording.  

## 1.3. Optional format

1.3.1. Use Conventional Commits if repository uses them.
