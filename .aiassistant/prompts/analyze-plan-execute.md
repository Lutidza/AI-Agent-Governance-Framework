# 1. Analyze -> Plan -> Execute

## 1.1. Purpose

1.1.1. Analyze the task first, then provide a concise implementation plan, then execute.

## 1.2. Constraints

1.2.1. Keep changes minimal and scoped.  
1.2.2. Do not modify unrelated files.  
1.2.3. Ask for confirmation before risky operations.  

## 1.3. Required output

1.3.1. Current state summary.  
1.3.2. Plan (3-6 steps).  
1.3.3. Risks and assumptions.  
1.3.4. Implementation.  
1.3.5. Verification checklist.
