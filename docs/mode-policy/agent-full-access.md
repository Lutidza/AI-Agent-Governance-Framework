# Agent (full access) — исключительный режим

## Статус

- режим не используется в ежедневной разработке
- режим разрешается только как отдельный управляемый сценарий

## Условия допуска

Допуск возможен только при одновременном выполнении всех условий:
1. Задачу нельзя корректно выполнить в `Read-only` или `Agent`.
2. Четко описан ожидаемый результат и границы действий.
3. Пользователь дал явное подтверждение на полный доступ.
4. Определен план отката в случае нежелательных изменений.

## Разрешено

- только действия, прямо перечисленные в подтвержденном плане

## Запрещено

- любые дополнительные инициативные действия вне согласованного списка

## Завершение режима

- после выполнения задача переводится обратно в `Read-only`
- фиксируется отчет: что сделано, где, с какими последствиями
