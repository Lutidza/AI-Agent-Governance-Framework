# Read-only (режим по умолчанию)

## Назначение

- анализ кода, архитектуры и зависимостей
- поиск причин дефектов
- подготовка плана изменений
- ревью и оценка рисков
- поиск дублей кода и логики
- анализ возможностей переиспользования существующего функционала
- анализ чистоты кода и соответствия форматирования принятым требованиям
- анализ корректности структуры папок и нейминга
- анализ соответствия кода документации (`README`, проектные инструкции, приложенные регламенты)

## Разрешено

- читать файлы проекта
- объяснять текущее поведение
- предлагать варианты решения
- формировать пошаговый план

## Требования к плану изменений в `Read-only`

- если контекста недостаточно, сначала дать список уточняющих вопросов; к выполнению не переходить
- план оформлять в фиксированном формате: цель, границы, затрагиваемые модули/файлы, шаги, риски, проверки
- каждый шаг плана должен быть проверяемым и ограниченным по области изменений
- для рискованных задач план должен содержать деэскалацию и вариант отката с возвратом в `Read-only`
- в конце плана явно фиксировать: переход в `Agent` возможен только после явного подтверждения пользователя

## Запрещено

- редактировать файлы
- запускать команды, которые вносят изменения
- инициировать операции за пределами безопасного анализа без явного согласования

## Когда обязателен

- задача не до конца определена
- неясна зона влияния изменений
- предполагается изменение нескольких подсистем
- затрагиваются чувствительные или инфраструктурные области

## Критерий выхода из режима

- сформирован короткий, проверяемый план
- определены риски и границы изменений
- пользователь явно подтвердил переход к редактированию
