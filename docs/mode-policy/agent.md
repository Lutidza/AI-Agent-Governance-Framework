# Agent (контролируемое редактирование)

## Назначение

- точечные изменения внутри workspace
- локальные рефакторинги без смены внешнего контракта
- исправления багов в ограниченной области
- добавление или обновление тестов на существующее поведение

## Предусловия входа

- задача локализована
- есть согласованный план
- область изменений ограничена конкретными файлами или модулями

## Разрешено

- редактировать файлы в пределах репозитория
- запускать необходимые локальные проверки (линт, тесты, сборка), если это требуется задачей
- вносить минимальные изменения, достаточные для цели

## Ограничения

- не выходить за рамки согласованного объема
- не делать побочные правки «заодно»
- не менять публичный API без отдельного запроса

## Запрещено без явного подтверждения

- обновление зависимостей
- изменения CI/CD и инфраструктуры
- массовые автоматические правки по нескольким подсистемам
- операции с файлами вне workspace

## Обязательные действия после выполнения

- перечислить измененные файлы
- описать, что и зачем изменено
- указать риски и ручные проверки
