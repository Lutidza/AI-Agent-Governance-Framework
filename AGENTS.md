# Контракт агентов проекта документации

Этот файл задает обязательные операционные правила для всех агентов, подключенных к репозиторию.

## Роль и границы ответственности

- Агент ОБЯЗАН поддерживать и развивать проект документации для ИИ как системный продукт.
- Агент ОБЯЗАН различать два рабочих контура:
  - `docs/` — универсальная документационная система для других проектов.
  - `.aiassistant/` — JetBrains-специфичный контур тонкой настройки среды.
- Агент ЗАПРЕЩЕНО смешивать назначение контуров в одном изменении без явного запроса пользователя.

## Область действия AGENTS.md

- Этот файл регулирует только работу агента, который создает и сопровождает документацию в данном репозитории.
- Документы из `docs/` являются целевым продуктом и описывают правила применения в других проектах.
- Правила из `AGENTS.md` и `docs/` не конкурируют, так как относятся к разным контурам.
- IF формулировки в `docs/` и текущие действия по их редактированию расходятся THEN агент ОБЯЗАН остановиться и запросить уточнение у пользователя.

## Правила написания документации

- Агент ОБЯЗАН писать в императивной форме.
- Агент ОБЯЗАН использовать нормативные маркеры: `ОБЯЗАН`, `ЗАПРЕЩЕНО`, `ДОПУСКАЕТСЯ`.
- Агент ОБЯЗАН использовать формат `IF -> THEN` для условных правил.
- Агент ОБЯЗАН формулировать правила как проверяемые условия.
- Агент ЗАПРЕЩЕНО использовать двусмысленные или оценочные формулировки.

## Правила изменения файлов

- Перед правками агент ОБЯЗАН определить целевой контур (`docs/` или `.aiassistant/`).
- Агент ОБЯЗАН изменять только файлы, согласованные пользователем в текущем шаге.
- Агент ЗАПРЕЩЕНО создавать новые файлы/разделы без явного запроса пользователя.
- Агент ЗАПРЕЩЕНО удалять обязательные разделы и правила без явного запроса.
- После правок агент ОБЯЗАН проверить и синхронизировать перекрестные ссылки.

## Стек-независимая архитектура документации

- Агент ОБЯЗАН поддерживать разделение:
  - ядро системы (роли, этапы, workflow, handoff, ограничения, MCP-подходы);
  - заменяемый стековый блок (технологический профиль конкретного проекта).
- Агент ДОПУСКАЕТСЯ изменять стековый блок без изменения ядра, если это явно задано пользователем.

## Мультиагентный режим

- IF одновременно работает несколько агентов THEN один агент ОБЯЗАН быть дирижером (оркестратором).
- IF агент не является дирижером THEN агент ОБЯЗАН работать только в рамках своей роли и handoff-контракта.
- IF завершен шаг THEN агент ОБЯЗАН передать краткий handoff: что изменено, почему, что проверять дальше.

## Язык и именование

- Имена файлов и папок: английский язык.
- Содержимое документации: русский язык, если пользователь не указал иное.
- Термины ролей и процессов ОБЯЗАНЫ быть единообразными во всех документах.
